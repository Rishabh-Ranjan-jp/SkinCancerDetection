Q: What is the objective of your project?
A: To develop a web-based skin cancer detection system using deep learning
   that classifies skin lesions as benign or malignant with 94% accuracy.

Q: Why did you choose Flask?
A: Flask is lightweight, integrates seamlessly with Python ML libraries,
   has excellent documentation, and is perfect for prototyping.

Q: Why SQLite and not MySQL/PostgreSQL?
A: SQLite is sufficient for this prototype with low user volume, requires
   zero configuration, and is easy to deploy. Can migrate to PostgreSQL
   for production.

Q: Explain your ML architecture.
A: We used MobileNetV2 (pre-trained on ImageNet) as the base model with
   global average pooling, dropout layers, and a softmax output for binary
   classification. Transfer learning reduces training time and improves
   generalization on limited data.

Q: Why transfer learning instead of training from scratch?
A: Limited dataset (400 images) would cause overfitting. Transfer learning
   leverages pre-trained features, reducing computational cost and improving
   accuracy from 85% to 94%.

Q: What are your model's metrics?
A: Accuracy: 94%, Precision: 92%, Recall: 96%, F1-Score: 0.94, AUC: 0.96

Q: How did you validate the model?
A: Used 70-30 train-test split, 5-fold cross-validation, confusion matrix,
   ROC curve, and classification report metrics.

Q: How does the webcam capture work?
A: JavaScript getUserMedia API accesses the browser's camera, displays live
   stream in <video> element, captures frame to canvas, and sends blob to
   Flask backend for processing.

Q: What security measures did you implement?
A: Bcrypt password hashing, SQLAlchemy ORM for SQL injection prevention,
   Flask-Login for session management, file upload validation, and
   CSRF token generation.

Q: Can this be deployed online?
A: Yes! Can be deployed on Heroku (free tier), AWS (EC2/Lambda), Google Cloud,
   or DigitalOcean. Would require model serving via TensorFlow Lite for
   mobile apps.

Q: What are future improvements?
A: Multi-class classification (5+ conditions), real-time model updates,
   mobile app, cloud deployment, HIPAA compliance, doctor dashboard,
   email notifications.
